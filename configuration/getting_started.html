Getting Started¶

Note

For all steps (Development,starting the Server and executing tests) the User must accept the Minecraft EULA, by own configured property!
by example setting a envierment property like export mc_accept_eula=true

Local Usage¶

For local Usage you can use the Vagrant file, located at the Provision Directory.

Start the Local Vagrant Machine¶
 cd provisioning
 vagrant up

After successfull starting you can join the server at localhost:25567

Note

The original gameport will mapped in the Vagrant file from 25565 to 25567.

connect the Local Vagrant Machine¶
   vagrant ssh
reinit the Local Vagrant Machine¶
   vagrant rsync && \
   vagrant provision

Advanced Environment Configuration¶

For usage with Hetzner Cloud you must prepare you system a little bit more.

export HCLOUD_TOKEN=$(pass internet/hetzner.com/projects/minecraft/terraform-token)

Configure Ansible¶

configure ansible python Environment¶
virtualenv
pip install ansible requests ansible-cmdb
ansible-galaxy install -f -r maintenance/requirements.yml

You must set your used private_key_file and some SSH Extra args -o IdentitiesOnly=yes, otherwise the fail2ban process will block your IP!.

example ansible.cfg¶
[defaults]
validate_certs=False
private_key_file=~/.ssh/id_ed25519

[ssh_connection]
ssh_extra_args=-o IdentitiesOnly=yes
